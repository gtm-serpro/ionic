FAB (Floating Action Button) ✓ Boas Práticas DS Gov
  • Um primário por tela: Destaque a ação principal
  • Ordem desktop: Terciário → Secundário → Primário (da esquerda para direita)
  • Ordem mobile: Empilhados, primário no topo
  • Labels claras: Verbos de ação ("Salvar", "Enviar", "Cancelar")
  • Ícones consistentes: Esquerda = ação imediata | Direita = continuidade
  • Área mínima: 44x44px para acessibilidade (WCAG 2.5.5)
  • Focus visível: Outline azul para navegação por teclado
  • Border radius: 24px (capsule/pílula)
  • Transições: 0.15s cubic-bezier(0.4, 0, 0.2, 1)

✓ Boas Práticas - Formulários DS Gov
  • Labels claros: Sempre use labels descritivos para cada campo
  • Campos obrigatórios: Marque com asterisco (*) vermelho
  • Mensagens de ajuda: Use textos auxiliares para orientar o preenchimento
  • Validação em tempo real: Mostre feedback imediato (sucesso/erro)
  • Ícones de status: Use ícones para indicar sucesso (✓) ou erro (⚠)
  • Agrupamento lógico: Organize campos em seções relacionadas
  • Layout responsivo: Use grid para organizar campos em múltiplas colunas
  • Mensagens de erro claras: Explique o que está errado e como corrigir
  • Placeholder útil: Use exemplos de formato esperado
  • Acessibilidade: Garanta navegação por teclado e leitores de tela
